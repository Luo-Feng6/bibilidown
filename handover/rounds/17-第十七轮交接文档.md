## 二、已推送改动详解

### 2.1 InfoTip 组件修复（SettingsPage.tsx）

**问题**：
- 气泡 `left: 50%` + `translateX(-50%)` 居中 → ⓘ 靠近左边缘时一半气泡超出视口被遮挡
- `whiteSpace: 'pre-wrap'` 但 `\n` 字面字符在某些情况下不换行
- 背景用 `var(--text-inverse)`，亮色主题下是深色字 + 深色底 = 看不清

**修复**：
- 定位改为 `left: 0`（左对齐），箭头 `left: 7px` 指向 ⓘ 圆心
- `{text}` 改为 `.split(/\n|\\n/).map()` → 每条用 `<br/>` 分割，彻底解决换行
- 颜色全部硬编码 `#1a1a2e` / `#e0e0e0`，不依赖 CSS 变量

### 2.2 重复视频重新排队（downloadStore.ts）

**问题**：用户把已完成的视频再加一次 → 提示"已在队列中" → 什么都不做 → 用户困惑

**修复**：`addItem()` 检测到重复时，如果现有项是 `completed` 或 `paused` → 重置为 `queued`（进度清零、速度还原），返回 `{ duplicate: false, reused: true }`。只有 `queued`/`downloading` 的才真正报重复。

### 2.3 下载方案（Preset）系统

**新增文件**：`src/stores/presetStore.ts`

**数据模型** — 一个 preset 保存 7 个下载相关设置：

| 字段 | 类型 | 默认值 |
|------|------|--------|
| `preferredQuality` | string | `'1080P60'` |
| `preferredFormat` | string | `'mp4'` |
| `preferredAudioFormat` | `'m4a'\|'m4s'\|'mp3'` | `'m4a'` |
| `downloadDanmaku` | boolean | `false` |
| `downloadSubtitle` | boolean | `false` |
| `filenameTemplate` | string | `'{title}_{quality}'` |
| `downloadModeStyle` | `'popup'\|'inline'` | `'popup'` |

**API**：`saveCurrentAsPreset` / `updatePreset` / `deletePreset` / `applyPreset` / `renamePreset`

**UI 位置**：
- **设置页顶部**：「下载方案」Section — 下拉选择 + 💾更新 + ✏重命名 + 🗑删除 + 💾保存当前
- **首页解析区**：「方案: xxx ▾」下拉 + 🔄 同步按钮

**不纳入 Preset 的设置**（跨方案通用）：主题、主题色、封面、路径、并发数、重试、通知、打开文件夹、面板宽度、托盘、登录

### 2.4 文件名模板变量插入（SettingsPage.tsx）

**问题**：用户要手敲 `{title}` `{bvid}` 等变量，容易记错

**修复**：输入框右侧 `+` 按钮 → 弹出变量备选面板（8 个按钮：标题/BV号/清晰度/UP主/日期/时间/格式/模式）→ 点击插入到光标位置（支持选中替换）

### 2.5 下载模式 hint 直白化

**问题**：原 hint 只写「弹窗 / 内联」，用户不读 ⓘ 不知道内联会直接下载

**修复**：hint 改为「弹窗模式每次确认 · 内联模式点击按钮即直接下载」

## 四、核心问题与解决方案

### 4.1 CSS 变量的坑（复现）

三轮前刚修完 `--surface-raised` 未定义，本轮又发现 `--text-inverse` 在亮色主题下是深色。InfoTip 组件两个地方都是硬编码 `#1a1a2e` / `#e0e0e0` 固定的。结论：**深色底+亮字的 UI 元素不要用 CSS 变量，直接写死**。

### 4.2 zustand persist 重复问题

`presetStore` 使用 `skipHydration: true`，必须在 `App.tsx` 的 `useEffect` 里手动 `rehydrate()`，否则和 userPrefsStore 一样会触发 zustand v5 的无限重渲染 bug。已按相同模式处理。

### 4.3 Phosphor 图标不支持 `title` 属性

`Copy` 图标直接写 `title="复制"` < TS 报错 → 改用外层 `<span title="...">` 包裹。

### 4.4 预设系统设计取舍

用户最初说的是"命名方案"，后续补充了"更新已有方案"和"首页同步"。最终设计：
- `activePresetId` 记录哪个预设被选中（null = 默认）
- 选中即 `applyPreset`，设置页其他控件 reactively 更新
- 切换方案后 danmaku/subtitle 自动同步（TogglePill 从 store 读），但已解析视频的 quality 选择不会变（那是 VideoCard 的本地 state）
