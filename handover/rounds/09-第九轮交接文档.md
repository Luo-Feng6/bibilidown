## 二、做了什么

### 已完成的 10 项工作

#### 1. ✅ Electron 下载进度消费（第八轮遗留）

- **问题**：`electron/main.cjs` 每 200ms 发出 `download:progress` 事件，但 `download-manager.ts` 从未注册监听器。Electron 模式下进度条、速度、speedHistory 全死。
- **修复**：`processDownloadElectron` 开头注册 `api.onDownloadProgress()`，按 `filePath` 匹配视频/音频流，合并双流进度，~200ms 节流更新 store。`try/finally` 确保监听器释放。

#### 2. ✅ Electron 视频+音频并行下载（第八轮遗留）

- **问题**：`processDownloadElectron` 中 Phase 1（视频）和 Phase 2（音频）串行 `await`，耗时翻倍。
- **修复**：改为 `Promise.all` 并行下载，错误统一处理。浏览器模式也同步改为并行。

#### 3. ✅ 暗色主题 hover 效果统一（第八轮遗留）

- **问题**：12 处硬编码 `rgba(255,255,255,0.06)`（暗偏）或 `rgba(0,0,0,0.06)`（亮偏），切换主题时 hover 不可见。
- **修复**：全部替换为 `var(--surface-overlay)`，覆盖 7 个组件文件（DownloadPanel、DownloadItem、FfmpegBanner、InputBar、LoginPanel、TitleBar、Toast、VideoCard）。
- **额外修复**：`TitleBar`/`VideoCard` 的 `onMouseLeave` 中 `color = ''` → 显式设回 `var(--text-tertiary)`/`var(--text-secondary)`。

#### 4. ✅ 打包测试

- `npm run dev:electron` — Electron 窗口正常打开，无报错。
- `npm run build:electron` — 成功打出 Windows NSIS 安装包 + 便携版（`release/` 目录）。
- 首次打包遇到 `EPERM: rename`（旧 `release/` 目录被锁），清理后重试通过。

#### 5. ✅ 表单字段警告修复

- **问题**：浏览器 Console 警告 "A form field element should have an id or name attribute"。
- **修复**：6 个 `<input>` 添加 `id` — InputBar（`bilibili-url-input`）、LoginPanel 短信码/密码/手机号、VideoCard 字幕/弹幕复选框。

#### 6. ✅ 暗色主题 body/html 白色背景

- **问题**：暗色模式下页面背景仍然是白色（从 `<body>` 透出），因为 `data-theme` 属性只设在 React 根 `<div>` 上，`body` 和 `html` 始终拿亮色 CSS 变量。
- **根因**：`App.tsx` 的 `data-theme` 设在了 `<div id="root">` 内，但 `body { background-color: var(--surface-root) }` 的 `--surface-root` 解析在祖先元素（`<html>`/`:root`）上下文，不在 `[data-theme]` 子树内。
- **修复**：① `App.tsx` useEffect 同步 `data-theme` 到 `document.documentElement`；② `index.css` 加 `html { background-color: var(--surface-root); }`。

#### 7. ✅ 暗色主题芯片可见性 + 选中态粉色

- **问题**：暗色模式下未选中芯片透明背景 + `rgba(255,255,255,0.08)` 边框 → 在深色背景上几乎看不见。
- **修复**（3 轮迭代）：
  - 第一轮：芯片加 `--chip-bg-rest: rgba(255,255,255,0.05)` + `--chip-border-rest: 0.10` + B 站粉选中态。
  - 第二轮：用户反馈底色被换掉 → 恢复原 slate 灰（`var(--gray-900)` 等），只覆盖芯片 token。
  - 第三轮：用户反馈未选中太暗 → 提亮到 `bg-rest: 0.08`、`border-rest: 0.14`、hover `0.12/0.22`。
- **最终效果**：未选中可见、悬停有反馈、选中 B 站粉醒目、不可用极淡灰。

#### 8. ✅ 设置页主题色选择器

- **新增**：设置页"外观"区域新增"主题色"一行，5 个圆点色块：蓝 / 粉 / 青 / 紫 / 金。
- **实现**：`userPrefsStore` 增 `accentColor` 字段（持久化）→ `App.tsx` useEffect 根据主题暗/亮设置 `--color-accent` 等 4 个 CSS 变量到 `document.documentElement`。

#### 9. ✅ 历史记录逐条删除 + 导入导出说明

## 四、关键问题深度分析

### 问题 1：暗色主题白屏 — CSS 变量作用域陷阱

```
错误链路：
  1. App.tsx 渲染 <div data-theme="dark">
  2. <html> 和 <body> 是 #root 的祖先，在 CSS 树中在上面
  3. body { background-color: var(--surface-root) } 解析 --surface-root 时
     寻找的是 <body> 自身或祖先上的 --surface-root 定义
  4. data-theme="dark" 在 <div> 上，body 看不到
  5. body 从 :root 拿值 → 永远亮色主题的 --surface-root = #F8FAFC (白色)
  6. 所有半透明组件（Mica 88% 不透明度）下透出白色底色
```

```
修复：
  useEffect(() => {
    document.documentElement.setAttribute('data-theme', theme)
  }, [theme])
  
  // 现在 data-theme="dark" 在 <html> 上
  // body、#root、所有子元素全部进入暗色 CSS 变量作用域
```

### 问题 2：主题色不生效 — CSS 优先级

```
错误链路：
  1. JS setProperty('--color-accent', '#FB7299') → 内联样式
  2. tokens.css [data-theme="dark"] { --color-accent: var(--accent-400) } → 样式表规则
  3. 同时 tokens.css @media { :root:not([data-theme="light"]) { --color-accent: ... } } → 第二条规则
  4. CSS 自定义属性在多种规则同时定义时，优先级取决于选择器特异性
  5. 两个样式表规则 + 一个内联样式 → 浏览器可能不按预期级联
```

```
修复：
  document.documentElement.style.setProperty('--color-accent', preset.main, 'important')
  // 第三个参数 'important' 确保内联样式优先级最高
```

### 问题 3：芯片可见性 — 亮色 token 在暗色背景下不可见

```
设计原则：
  - 亮色模式芯片：透明底 + 灰色边框 (#E2E8F0)，在白卡片上显眼
  - 暗色模式芯片：同样的透明底 + rgba(255,255,255,0.08) 边框，在深色 (#0F172A) 上几乎看不见
  
解决：
  - 暗色覆写芯片 token（在 [data-theme="dark"] 和 @media 两处同步）
  - bg-rest: 8% 白 → 可见填充
  - border-rest: 14% 白 → 可见边框
  - hover: 12% bg + 22% border → 清晰的交互反馈
  - selected: 跟主题色（--color-accent）→ 用户可选蓝/粉/青/紫/金
```

## 六、暗色主题 Token 架构

```
tokens.css 三层架构：

Layer 1 — Primitives (不变)
  :root { --gray-900: #0F172A; --brand-pink: #FB7299; ... }

Layer 2 — Semantics (随主题切换)
  :root, [data-theme="light"] { --surface-root: var(--gray-50); ... }
  [data-theme="dark"]        { --surface-root: var(--gray-900); ... }
  @media (prefers-color-scheme: dark) { ... }

Layer 3 — Components (静态默认值)
  :root { --chip-bg-rest: transparent; ... }

暗色覆写 (在 Layer 2 dark 段内追加):
  [data-theme="dark"] {
    --chip-bg-rest: rgba(255,255,255,0.08);
    --chip-border-rest: rgba(255,255,255,0.14);
    --chip-bg-selected: var(--color-accent);  ← 跟随用户选择的主题色
    ...
  }

主题色注入 (JS, 在 App.tsx useEffect):
  document.documentElement.style.setProperty('--color-accent', '#FB7299', 'important')
```
