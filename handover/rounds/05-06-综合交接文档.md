## 二、做了什么（8 项功能）

### 2.1 ml 收藏夹分页加载

**问题**：
```
parseMediaList 仅调 pn=1 → 返回 20 条
多余视频只有 console.log 提示，用户无感知
```

**方案**：
```
首次解析 → 首页 20 条 + 存分页状态到 parseStore
EpisodeList 底部显示"加载更多"按钮
点击 → loadMoreMediaList() → pn++ → appendVideos → 直到全部加载完
```

**改了什么**：

| 文件 | 变更 |
|------|------|
| `stores/parseStore.ts` | +4 字段（mlId/mlPage/mlTotalCount/isLoadingMore）+ setMlPagination/setLoadingMore；reset() 清除 |
| `services/bilibili-api.ts` | 提取 mlHeaders() + mediasToVideos() 复用；parseMediaList 存分页信息；+loadMoreMediaList() |
| `components/EpisodeList.tsx` | +hasMore/isLoadingMore/onLoadMore props；列表底部"加载更多"按钮（含 Spinner loading 态） |
| `App.tsx` | 读 mlId/mlTotalCount，计算 mlHasMore，传 props + 接线 onLoadMore |

**分页判断**（容错设计）：
```
优先: data.has_more（API 返回值）
Fallback: medias.length >= 20 && videos.length < totalCount
```

**用户体验**：每页加载完 Toast "已加载 40 / 120 个视频"；全部加载完按钮自动消失。

### 2.2 ml cid 懒解析

**问题**（§6.12 已知限制）：
```
medialist API 只返回 bvid，不含 cid
用户加入下载队列 → item.cid = undefined → 下载报"缺少 cid"
```

**方案**：
```
加入队列前检测 cid == null + bvid 存在
  → 调 resolveCidForBvid(bvid) → GET /x/player/pagelist
  → 返回第一 P 的 cid
  → 补齐后再加入队列
批量场景: Promise.allSettled 并行解析，失败跳过不阻塞成功的
```

**改了什么**：

| 文件 | 变更 |
|------|------|
| `services/bilibili-api.ts` | +resolveCidForBvid(bvid) — 调 /x/player/pagelist，返回 pages[0].cid |
| `App.tsx` | handleAddToQueue async：cid 缺失时 lazy resolve；handleBatchAddToQueue async：Promise.allSettled 并行 + 统计 added/skipped |

**单视频**: 解析失败 → Toast error → 不加入队列  
**批量**: 成功的加入队列、失败的 Toast warning → "N 个视频无法获取下载信息，已跳过"

| `pages/HistoryPage.tsx` | **重写** — 51行→290行。Header + 搜索栏 + 筛选芯片 + 列表（标题/BV/画质/大小/时间/状态/重新下载/删除）+ 双态空状态 |
| `App.tsx` | +handleReDownload(bvid) — navigate('download') + parseBilibiliUrl + Toast |

### 2.5 FFmpeg 检测与警告横幅

**问题**：
```
用户系统未装 FFmpeg → 下载完成但合并失败
错误只出现在下载面板的失败消息里，不够醒目
```

**方案**：
```
应用启动时检测 FFmpeg（Electron IPC → main.cjs 查找逻辑）
缺失时下载页顶部显示黄色警告横幅
提供: 下载 FFmpeg（外链）+ 已安装重新检查 + 关闭
```

**新增文件**：

| 文件 | 职责 |
|------|------|
| `stores/ffmpegStore.ts` | 状态机：checking → available / missing / error；Electron IPC 调 main.cjs checkFfmpeg |
| `components/FfmpegBanner.tsx` | 黄色警告横幅，仅在 `status === 'missing'` 时显示，关闭后本次会话内不再出现 |

**接入口**：`App.tsx` 启动时 `useFfmpegStore.getState().checkFfmpeg()`，渲染 `<FfmpegBanner />` 在 InputBar 上方。

### 2.6 Cookie 启动验证

**问题**：
```
Cookie 持久化了但用户不知道有没有过期
过期 Cookie 导致高清画质不可用但没有提示
```

**方案**：
```
启动时调 validateCookie() → GET /x/web-interface/nav
  code=0 + isLogin=true  → valid
  code=-101（未登录）     → expired
  网络错误               → 保留 valid（不误判）
StatusBar 显示: 会话已过期（黄色按钮）/ 用户名
```

**改了什么**：

| 文件 | 变更 |
|------|------|
| `services/bilibili-api.ts` | +validateCookie() — 轻量验证，网络容错设计 |
| `stores/userPrefsStore.ts` | +cookieStatus（4态）/cookieUsername（非持久化，仅运行时） |
| `components/StatusBar.tsx` | 读 cookieStatus 显示过期提示；读 cookieUsername 显示用户名 |
| `App.tsx` | 启动时 await validateCookie() → setCookieStatus + setCookieUsername |

**容错哲学**: 网络错误 ≠ Cookie 过期。`validateCookie()` 在 fetch 失败或超时时返回 `valid: true`，避免误判导致用户不必要的重新登录。

## 四、遇到的问题与解决方案（12 项）

按发现时间排列：

### 4.1 动态 import Vite 警告（信息性）
- **现象**: build 输出 parseStore/toastStore 既动态 import 又静态 import 的 warning
- **原因**: bilibili-api.ts 用 `await import()` 避免循环依赖，但 App.tsx 又静态引用同 store
- **解决**: 无需处理。信息性警告，动态 import 语义正确

### 4.2 EmptyState Icon 类型不兼容
- **现象**: `TS2322` — Phosphor IconProps.size 类型为 `string|number|null|undefined`，不兼容 `number`
- **解决**: 改用 `import type { Icon as PhosphorIcon }` 替代 `React.ComponentType<...>`

### 4.3 hasMore 判断容错
- **问题**: medialist API 的 `data.has_more` 不一定存在（API 版本差异）
- **解决**: 双重判断 — 优先 API 返回值，fallback 数学计算

### 4.4 批量 cid 解析部分失败
- **问题**: `Promise.all` 一个失败全失败
- **解决**: 换 `Promise.allSettled` + 单独统计 added/skipped + 分别 Toast

### 4.5 历史记录存储上限
- **问题**: localStorage ~5-10MB，无限存储可能溢出
- **解决**: 500 条硬上限（~100KB），addEntry 自动截断

### 4.6 tsc 被从 build:electron 移除
- **现象**: Electron 打包子代理因当时 HistoryPage TS 错误，移除了 `tsc &&`
- **原因**: 子代理运行时 EmptyState Icon 类型错误尚未修复
- **解决**: 我的 session 修复了 TS 错误后，手动恢复 `tsc &&` 到 build:electron

### 4.7 Cookie 验证网络容错
- **问题**: 启动时网络未就绪 → `validateCookie()` 报错 → 误判 Cookie 过期
- **解决**: fetch 失败/超时返回 `valid: true` + console.warn，不过度反应

### 4.8 FFmpeg 查找路径不足
- **问题**: 打包后 FFmpeg 在 `resources/ffmpeg/` 但 main.cjs 没查这个路径
- **解决**: 补充 `path.join(process.resourcesPath, 'ffmpeg', 'ffmpeg.exe')` 作为第 4 步回退

### 4.9 关闭窗口 ≠ 退出应用
- **问题**: 用户关闭窗口 → 应用退出 → 下载中断
- **解决**: main.cjs close 事件改为 `event.preventDefault() + mainWindow.hide()`；托盘菜单提供"退出"；增加 `isQuitting` 状态区分

### 4.10 构建镜像下载慢
- **问题**: 国内网络从 GitHub 下载 Electron 二进制极慢
- **解决**: build:electron 加 `ELECTRON_MIRROR` + `ELECTRON_BUILDER_BINARIES_MIRROR` 指向 npmmirror

### 4.11 托盘图标缺失
- **问题**: 打包后没有托盘图标文件
- **解决**: main.cjs 尝试读 `tray-icon.png`；fallback 用硬编码的 16×16 粉色 PNG data URL

### 4.12 medialist API 需要登录
- **问题**: 没登录 Cookie 时调收藏夹 API 返回 -101
- **解决**: parseMediaList/loadMoreMediaList 的 headers 注入 globalCookie；未登录时错误信息提示"需要登录后查看"
