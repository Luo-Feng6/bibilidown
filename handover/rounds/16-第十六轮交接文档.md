## 四、核心问题与解决方案

### 4.1 下拉菜单背景透明（最隐蔽的 bug）

```
用户：下拉和下面文字重合
↓
排查：下拉有 backgroundColor: 'var(--surface-raised)'
↓
grep 全项目 → --surface-raised 在 tokens.css 中未定义
↓
CSS var() 解析失败 → 回退到 transparent
↓
修复：硬编码 #1a1a2e，和 InfoTip 气泡统一风格
```

这个 bug 影响了所有 SettingsSelect 下拉和触发按钮的打开态。开发者看不出——因为浏览器 DevTools 可能缓存了旧值或默认给了其他背景。

### 4.2 ⓘ 气泡暗色主题正常、亮色主题看不清

```
InfoTip 用 var(--text-inverse, #e0e0e0)
↓
tokens.css 中 --text-inverse 有定义：
  暗色主题: var(--gray-0) = #FFFFFF ✓
  亮色主题: var(--gray-900) = #0F172A ✗
↓
亮色下深字(#0F172A) + 深底(#1a1a2e) = 看不清
↓
修复：不用 CSS 变量，直接硬编码 #e0e0e0（亮灰）
```

### 4.3 弹窗没有取消按钮

```
历史原因：showDownloadChoiceDialog 是原生 DOM 弹窗
（非 React），设计时认为「总得选一个」
↓
用户合理诉求：我点错了想取消
↓
方案：
  1. 每个对话框加 ✕ 按钮（右上角，data-action="close"）
  2. 点遮罩也取消
  3. 返回值加 undefined → 4 个调用方都加 if (!mode) return
```

### 4.4 设置 → 下载链路是否接线

用户担心改了设置没用。验证结论：**已正确接线**。

- `preferredFormat` → `download-manager.ts:420`（Electron）和 `:837`（浏览器）
- `preferredAudioFormat` → `:421-422`（Electron）和 `:838-839`（浏览器）
- `filenameTemplate` → `:417-418`（Electron）和 `:735-736`（浏览器）
- `downloadPath` → 仅 Electron 的 `showSaveDialog` 用路径（浏览器无法控制下载位置）

### 关键数据流

```
设置页改 video/audio format
  → userPrefsStore.setPreferredFormat / setPreferredAudioFormat
  → 持久化到 localStorage
  → 下载时 download-manager 读 getState().preferredFormat / preferredAudioFormat
  → Electron: videoExt / audioExt 控制 showSaveDialog 默认扩展名
  → 浏览器: videoExt / audioExt 控制 triggerBrowserDownload 的文件名

设置页改 filenameTemplate
  → 下载时 applyFilenameTemplate(template, item)
  → {title}{bvid}{quality}{up}{date}{time}{format}{mode} 8 变量替换
  → sanitizeFileName → safeName → 最终文件名

弹窗模式 → showDownloadChoiceDialog
  → ✕ / 遮罩 → resolve(undefined) → if (!mode) return → 取消下载
  → 选按钮 → resolve('video-only'|...) → 加入队列
```
