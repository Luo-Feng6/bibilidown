## 二、关键 Bug 修复

### 2.1 退出登录后仍显示账号名

**现象**：点击「退出账号」→ 头像消失但名字还在 → 再点击仍有「更换账号」「退出账号」菜单

**根因**：
```typescript
// userPrefsStore.ts — 修复前
clearLoginInfo: () => set({ cookieStr: '', loginName: '', loginFace: '' }),
//                                              ↑ 没清 cookieUsername！

// StatusBar.tsx — 显示逻辑
const displayName = cookieUsername ?? loginName
//                   ↑ 优先读 cookieUsername，没清 = 永远有值
```

**修复**：
```typescript
clearLoginInfo: () => set({
  cookieStr: '', loginName: '', loginFace: '',
  cookieUsername: null,   // ← 新增
  cookieStatus: 'unknown', // ← 新增，防止显示「会话已过期」
}),
```

### 2.2 桌面快捷方式编码问题（3 次迭代）

| 版本 | 问题 | 修复 |
|------|------|------|
| v1 | Write 工具 → LF 换行 | cmd 不认 LF |
| v2 | bash printf → `\a` 被转义成响铃符（`D:\all app` → `D:ll app`） | 改用 Write 工具 |
| v3 | Write 工具 → LF 换行（无 CR） | `sed -i 's/$/\r/'` 转换 |
| v4 | 端口占用 + 无诊断 | 加 taskkill + 诊断提示 |
| v5 | 提示文案不诚实 | 注明需 Node.js |

最终正确的写法：`Write` 工具写内容 → `sed` 转 CRLF → 加 `chcp 65001` 支持中文 → 加 `taskkill` 清端口。

### 2.3 isElectron() 4 处重复

**修复前**：
```
TitleBar.tsx       → function isElectron() { return !!window.electronAPI }
SettingsPage.tsx   → function isElectron() { return !!(window as any).electronAPI }
download-manager.ts → function isElectron() { return !!window.electronAPI }
App.tsx            → const isElectron = !!electronVersion
```

**修复后**：全部引用 `src/utils/env.ts` 的 `isElectron()`。

## 五、设计决策记录

| 决策 | 选了什么 | 为什么 |
|------|---------|--------|
| AboutPage 位置 | `src/pages/AboutPage.tsx` | 与其他页面一致，方便 import |
| `isElectron()` 统一入口 | `src/utils/env.ts` | 避免 4 处重复，单一真相来源 |
| 文档拆分 | 4 份（README+ARCHITECTURE+CONTRIBUTING+CHANGELOG） | 首页简洁，详细内容按需展开 |
| 桌面快捷方式 | `.bat` 而非 `.lnk` | 纯文本可编辑，不依赖 Windows COM |
| 快捷方式编码 | UTF-8 BOM + CRLF + chcp 65001 | 中文不乱码，cmd 可执行 |
| 端口清理策略 | 启动时 taskkill 5173 | 上次 Vite 残留导致「打不开」 |
| `clearLoginInfo` | 清 5 个字段（含 cookieUsername + cookieStatus） | 退出登录后不残留任何登录态 |
| 浏览器版描述 | 诚实写"需 Node.js" | 不以"无需安装"误导用户 |
