## 二、核心修复：折叠下载面板数字徽章 writingMode 问题

### 2.1 用户反馈

> "在我的页面，下载主页的右边，把下载列表缩起来后，下面有个 2，显示的是没完成的吧，也就是未进行的，但为什么那个字是横着的"

### 2.2 现象

下载面板（DownloadPanel）折叠后是一个 36px 宽的窄竖条。展开按钮下方有数字徽章，显示正在下载/失败的任务数量。用户看到数字 "2" 是横着显示的，感觉违和。

### 2.3 根因分析

问题代码在 `src/components/DownloadPanel.tsx` 折叠状态的渲染中：

```tsx
{/* Badge counts */}
{summary.active.length > 0 && (
  <span style={{
    marginTop: '6px', fontSize: '10px', fontWeight: 600,
    color: 'var(--color-accent)',
    writingMode: 'vertical-rl',  // ← 问题在这里
  }}>
    {summary.active.length}
  </span>
)}
```

**CSS `writing-mode: vertical-rl` 的行为**：

- 改变的是**文字排版方向**（从上到下、从右到左），不是字符旋转
- 对 CJK 字符（中文/日文/韩文）有效：每个字自然地竖向排列
- 对**阿拉伯数字（0-9）和拉丁字母无效**：这些字符在竖排中保持 upright（直立、不旋转），由 `text-orientation: mixed`（默认值）决定
- 单个数字 "2" 只有一个字符，`writing-mode` 不影响它——看起来和普通横排 "2" 完全一样
- 在 36px 宽的窄竖条中，一个横着的 "2" 显得格格不入

### 2.4 解决方案

**去掉 `writingMode: 'vertical-rl'`**。

- 10px 字体大小的数字在 36px 宽的列中完全放得下
- 不再需要竖排效果（单数字也没法竖排）
- 显示更自然、更干净

同时去掉了失败数量徽章上的相同属性（`summary.failed.length`）。

### 2.5 诊断经验总结

> **CSS 知识点**：`writing-mode: vertical-rl` 不会旋转阿拉伯数字。如果需要旋转数字（如侧边栏标签），应使用 `transform: rotate(-90deg)`。对于单字符数字徽章，最简单的方案是去掉 writing-mode，直接横排显示。
