## 三、逐任务详解

### 3.1 任务1 — Electron API_BASE 修复

**问题根因**

项目中有 4 个服务文件各自定义了 `API_BASE = '/api/bilibili'`：
- `bilibili-api.ts` / `cookie-manager.ts` / `wbi-sign.ts` / `login-service.ts`

开发模式下，页面由 `http://localhost:5173` 加载，Vite 的 `server.proxy` 把 `/api/bilibili/*` 转发到 `https://api.bilibili.com`，一切正常。

但在 Electron **生产模式**下，页面由 `file://` 协议加载，`fetch('/api/bilibili/...')` 被解析为 `file:///api/bilibili/...`，不存在，请求失败。

**解决方案**

新建 `src/services/api-base.ts`，根据运行环境自动选择 URL：

```typescript
// 生产 Electron（file://）→ 绝对 URL
// 开发模式 / 浏览器 → Vite 代理路径
export const API_BASE = (isElectron() && !import.meta.env.DEV)
  ? 'https://api.bilibili.com'
  : '/api/bilibili'
```

4 个服务文件改为 `import { API_BASE } from './api-base'`，删除本地定义。`PASSPORT_BASE` 同理。

**效果**：开发模式不受影响（继续走 Vite 代理），生产 Electron 直接请求 `https://api.bilibili.com`。

**涉及文件**：
- 🆕 `src/services/api-base.ts`
- ✏ `src/services/bilibili-api.ts`（删本地 API_BASE）
- ✏ `src/services/cookie-manager.ts`（删本地 API_BASE + PASSPORT_BASE）
- ✏ `src/services/wbi-sign.ts`（删本地 API_BASE）
- ✏ `src/services/login-service.ts`（删本地 PASSPORT_BASE）

### 3.2 任务2 — 下载弹窗 React 化

**改造前**

`showDownloadChoiceDialog()` 和 `showMergeWarningDialog()` 两个函数约 160 行，全用 `document.createElement` + `innerHTML` 手拼 DOM：
- 无动画（瞬间出现/消失）
- 无键盘支持（Escape 无效）
- 样式写死在 JS 字符串中，难以维护
- 和项目已有的 `ConfirmDialog` React 弹窗风格不统一

**改造后**

新建 `src/components/DownloadChoiceDialog.tsx`（422 行），标准 React 组件：

- **三种状态**：单视频流（一键下载）/ DASH 双流（4 个选择按钮）/ 合并警告子页面
- **动画**：淡入 (`dialog-fade-in`) + 缩放 (`dialog-scale-in`)，和 `ConfirmDialog` 一致
- **键盘**：Escape 关闭
- **遮罩**：点击灰色区域关闭
- **主题**：全部使用 `var(--surface-elevated)`、`var(--color-accent)` 等 CSS 变量，跟随明暗主题
- **图标**：Phosphor icons（VideoCamera / MusicNotes / Archive / LinkSimple）

调用方式统一到 `dialog-service.ts`：
```typescript
// 之前：从 download-manager import
import { showDownloadChoiceDialog } from './services/download-manager'
// 之后：从 dialog-service import（和 showConfirm 同一入口）
import { showDownloadChoice } from './services/dialog-service'
```

**架构**：
```
dialog-service.ts (showDownloadChoice)
  → createRoot + React.createElement
    → DownloadChoiceDialog.tsx (纯展示组件)
      → onSelect / onCancel 回调
```

**涉及文件**：
- 🆕 `src/components/DownloadChoiceDialog.tsx`
- ✏ `src/services/dialog-service.ts`（+65 行，新增 `showDownloadChoice`）
- ✏ `src/App.tsx`（3 处调用改为新 API）
- ✏ `src/services/download-manager.ts`（删 ~160 行旧函数 + 1 处 fallback 改为新 API）

## 四、过程中遇到的问题与解决

### 4.1 问题：下载点按钮报 404

**现象**：四任务完成后，用户测试发现点击下载按钮报 404 错误，桌面端和浏览器端都受影响。

**排查过程**：
1. 确认所有模块编译通过（Vite HTTP 200）
2. 确认 API 代理正常工作（curl B站 API 返回正常）
3. 确认 CSS 无语法错误（花括号匹配）
4. 逐模块检查 import 链
5. 发现 DownloadChoiceDialog 使用了 `LinkSimpleHorizontal` 图标——虽然包里有这个导出，但在 Vite 按需加载时可能解析失败，导致组件渲染时抛错，进而整个下载链路中断

**修复**：`LinkSimpleHorizontal` → `LinkSimple`（更通用的图标名，兼容性更好）。

---

### 4.2 问题：应用整体颜色丢失

**现象**：用户反馈"我原来的那些颜色呢"，应用界面配色异常。

**排查**：检查 DownloadChoiceDialog 发现两处问题：
1. 警告横幅使用了 `--color-warning-text` 和 `--color-warning-border` 这两个**不存在于项目 tokens.css** 的 CSS 变量，fallback 到硬编码的琥珀色 `#92400e` / `#f59e0b`，在暗色主题下极不协调
2. 组件内置了重复的 `<style>` 标签注入 keyframes（与 index.css 全局定义重复）

**修复**：
- 警告横幅改用项目 tokens：`--text-secondary` + `--border-subtle`
- 删除组件内重复的 `<style>` 标签

---

### 4.3 问题：关于页图标不显示

**现象**：AboutPage 中 `<img src="/favicon.svg">` 不显示。

**修复**：添加 `onError` fallback 到 `favicon.png`，同时更新版本号 v7.7.0→v7.7.2，补全更新日志。

---

### 4.4 问题：Electron 启动时 `cross-env` 找不到

**现象**：直接在 bash 中跑 `cross-env ... electron .` 报 `command not found`。

**原因**：`cross-env` 安装在 `node_modules/.bin/` 下，npm scripts 自动把该目录加入 PATH，但直接 bash 调用不会。这不是 bug，只是诊断时需要手动设置环境变量。

**解决**：用 `export VAR=value` 替代 `cross-env`，或直接用 `npm run dev:electron` 即可。
