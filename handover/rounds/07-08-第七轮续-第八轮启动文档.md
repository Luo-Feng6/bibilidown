### 1.2 Electron 打包修复（Windows Defender EPERM）

**问题**：`npm run build:electron` 在 electron-builder 提取阶段报 `EPERM: operation not permitted, rename 'win-unpacked.tmp' -> 'win-unpacked'`，重试 5 次（每次 2-10s 延迟）全部失败，连 `cp -r` 出来的目录都无法 `rm -rf`。

**根因**：Windows Defender 实时扫描在 Electron 二进制（`electron.exe` 225MB、`dxcompiler.dll` 25MB 等）刚解压到 `.tmp` 时就锁定文件，导致 Node.js `fs.rename()` 被拒绝。

**解决方案**：patch `node_modules/app-builder-lib/out/util/electronGet.js` 的 `extractArchive()` 函数——rename 失败时 fallback 到 `fs.cp + fs.rm`（拷贝比 rename 慢但不受 Defender 锁影响）。

```js
// 原始: await fs.rename(tmpDir, dir);
// Patch: try rename → on EPERM → fs.cp(tmpDir, dir) + fs.rm(tmpDir)
```

**打包结果**：
```

release/BilibiliDown 7.0.0.exe          (99 MB) — 便携版
release/BilibiliDown Setup 7.0.0.exe.blockmap — 增量更新
```
