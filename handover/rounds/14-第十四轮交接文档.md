## 四、核心问题与解决方案

### 4.1 下载完成无法定位文件

**问题链**：
```
用户下载完成 → 点标题想看文件在哪 → 什么都没发生
↓
原因：App.tsx 没有 handleOpenFolder 实现，DownloadPanel 的 onOpenFolder 传入但没有 handler
```

**解决方案**：
- Electron：新增 `shell:showItemInFolder` IPC handler → preload 暴露 → App.tsx 调用
- 浏览器：无法打开文件夹，改为 `navigator.clipboard.writeText(filename)` + Toast "已复制文件名"

**关键技术点**：下载完成时在 `store.updateItem` 中保存 `savedPath`（Electron 为完整路径，浏览器为文件名）

### 4.2 浏览器"合并"的困境

**问题**：用户点击"合并"但浏览器无 FFmpeg，无法真正合并音视频。

**解决方案**：
- 浏览器点"合并" → 弹出引导对话框
- 说明三种方案：①用桌面版 ②FFmpeg 命令 ③下载两个文件手动合并
- 用户选择"下载两个文件"→ 按 separate 模式下载
- Electron 点"合并"→ FFmpeg 自动合成

**FFmpeg 命令提示**（显示在弹窗和下载完成信息中）：
```bash
ffmpeg -i 视频.mp4 -i 音频.m4a -c copy 输出.mp4
```

### 4.3 清晰度 vs 下载方式的功能重叠

**问题**：清晰度芯片有"仅音频"选项，弹窗/内联按钮也有"仅音频"。用户困惑"为什么有两个地方选音频"。

**解决方案**：
- 清晰度芯片只保留画质选项（4K/1080P60/1080P/720P/480P）
- 下载方式（弹窗或内联按钮）负责视频/音频/合并的选择
- 各管各的，不再重叠

### 4.4 分别下载 404

**问题**：用户选"分别下载"时报 HTTP 404，但"仅视频"正常。说明某个流的 CDN 链接失效。

**根因**：`resolveDownloadUrl` 返回的 `DownloadUrlResult` 只有主 URL（`videoUrl`/`audioUrl`），没有备用链接。B站 API 响应中每个 DASH segment 都有 `backup_url` 数组，但代码丢弃了。

**解决方案**：
1. `DownloadUrlResult` 新增 `videoBackups: string[]` 和 `audioBackups: string[]`
2. `resolveDownloadUrl` 填充备用链接
3. `tryDownloadWithFallback` 函数：`[主链接, ...备用链接]` 逐个尝试，全部失败才降级为 `<a>` 直接下载

### 4.5 重复检测太严格

**问题**：用户下过 720P 仅视频，想再下 1080P 或仅音频，被重复检测拦截。

**根因**：`addItem` 的重复检测只比较 `bvid + cid`。

**解决方案**：增加 `quality + downloadMode` 比较。只有 **四个字段完全相同**（同 bvid、同 cid、同清晰度、同下载模式）且状态非 failed 时才拦截。

### 4.6 字幕弹幕开关的可见性

**问题**：`downloadDanmaku`/`downloadSubtitle` 只在设置页，用户不知道有这个功能。

**解决过程**：
1. 第一版：在 App.tsx 视频列表上方加独立行（被用户吐槽"浮在上面"）
2. 第二版：移到视频列表下方（被用户吐槽位置不好）
3. 最终版：移到每个 VideoCard 内部，内联模式下载方式按钮下面一行

**最终布局**：
```
清晰度: [4K] [1080P60] [1080P] [720P] [480P]
下载方式: [📹仅视频] [🎵仅音频] [📦分别下载] [🔗合并]    ⚙更多设置→
附带:     [💬弹幕] [📝字幕]
```

### 关键数据流（本轮修复后）

```
用户粘贴 BV 号 → 剪贴板检测 → InputBar
  ↓
解析 API → preferredQuality 默认选中 → VideoCard 显示
  ↓
清晰度选择（纯画质，无"仅音频"）
  ↓
┌─ 弹窗模式: 点「加入下载队列」→ 弹窗四选一 → addItem({ downloadMode })
│   ├─ 浏览器点"合并"→ 弹出 FFmpeg 引导 → 选"下载两个文件"→ mode='separate'
│   └─ Electron 点"合并"→ mode='merge'
│
└─ 内联模式: 直接点 [📹仅视频] [🎵仅音频] [📦分别下载] [🔗合并]
    └─ 附带开关: 💬弹幕 📝字幕
  ↓
addItem → 重复检测 (bvid+cid+quality+downloadMode) → 入队
  ↓
processDownload → resolveDownloadUrl → 获得主链接 + 备用链接
  ↓
tryDownloadWithFallback → [主URL, 备用1, 备用2, ...] 逐个尝试
  ↓
下载完成 → savedPath 记录 → 弹幕/字幕检查 → history 记录
  ↓
用户点已完成标题:
  ├─ Electron → shell.showItemInFolder(savedPath)
  └─ 浏览器 → navigator.clipboard.writeText(filename) + Toast
```
