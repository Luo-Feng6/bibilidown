### 2.2 主题色全链路覆盖

**核心问题**：CSS 变量 `--color-accent` 是 hex（如 `#FB7299`），但 shadows/rings/glows 需要 `rgba()`，无法直接在 CSS 里 `rgba(var(--color-accent), 0.12)`。

**解决方案**：新增 `--color-accent-rgb` 变量，值为纯 RGB 分量（如 `251, 114, 153`），在 `App.tsx` useEffect 中随主题色同步注入。CSS 侧用 `rgba(var(--color-accent-rgb, fallback), alpha)` 模式。

**受影响的 CSS 变量**（tokens.css）：
| 变量 | 改动前 | 改动后 |
|------|--------|--------|
| `--shadow-ring` | `rgba(59,130,246,0.12)` 硬编码蓝 | `rgba(var(--color-accent-rgb, 59,130,246), 0.12)` |
| `--shadow-glow` | 同上 | 同上 |
| `--input-ring-focus` | 硬编码蓝 | 跟随主题色 |
| `--progress-glow` | 硬编码蓝 | 跟随主题色 |
| `--chip-bg-selected` | `var(--brand-pink)` | `var(--color-accent)` |
| `--chip-border-selected` | `var(--brand-pink)` | `var(--color-accent)` |

**受影响的组件**：
| 组件 | 改动前 | 改动后 |
|------|--------|--------|
| `EpisodeList.tsx` | `rgba(59,130,246,0.06)` 硬编码 | `var(--color-accent-muted)` |
| `Toast.tsx` (info) | `rgba(59,130,246,0.08)` 硬编码 | `var(--color-accent-muted)` |
| `index.css` body gradient | `rgba(59,130,246,0.03)` 硬编码 | `rgba(var(--color-accent-rgb,...), 0.03)` |

## 三、关键问题与解决方案

### 3.1 `--color-accent-rgb` 的设计

**问题**：CSS 中 `rgba()` 需要 RGB 分量，但 `--color-accent` 是 hex 字符串（如 `#FB7299`）。CSS 不支持 `rgba(hex, alpha)`，也不支持颜色空间转换。

**尝试过的方案**：
1. 每个使用场景单独维护 rgba 值 → 太分散，维护噩梦
2. 用 `color-mix()` → 浏览器兼容性不够好（Chromium 111+）
3. 定义 `--color-accent-rgb` 为分量值 → ✅ 最终采用

**实现**：
- `App.tsx` useEffect 中 `root.setProperty('--color-accent-rgb', preset.rgb, 'important')`
- CSS 侧：`rgba(var(--color-accent-rgb, 59, 130, 246), 0.12)` — fallback 值是蓝色 RGB 分量，保证 JS 注入前首次渲染不闪白
- 所有 shadows/rings/glows/focus-rings/progress-glow 统一改为这个模式

### 3.2 LoginPrompt 无限重渲染

**错误**：`Maximum update depth exceeded` in `<LoginPrompt>`

**根因**：`useUserPrefsStore((s) => ({ loginName: s.loginName, loginFace: s.loginFace }))` 每次返回新对象引用，zustand 检测到"变化"触发重渲染，重渲染又产生新对象 → 死循环。

**修复**：拆成两个独立 selector（原始类型不会触发引用比较问题）：
```tsx
const loginName = useUserPrefsStore((s) => s.loginName)
const loginFace = useUserPrefsStore((s) => s.loginFace)
```

**教训**：zustand 中 selector 返回对象必须用 `useShallow`；返回原始类型可以直接用。

### 3.3 亮色模式主题色选中圆圈发黑

**问题**：SettingRow 中 `border: prefs.accentColor === c.key ? '2px solid var(--text-primary)' : …`，亮色下 `--text-primary` = `#0F172A`（近乎黑色），在白色背景上看起来是黑框套色块。

**修复**：改用 `outline` + `outlineOffset`：
```tsx
outline: selected ? '2px solid var(--color-accent)' : '2px solid transparent',
outlineOffset: '2px',
transform: selected ? 'scale(1.15)' : 'scale(1)',
```

**效果**：选中色块放大 15% + 同色 outline，亮暗都自然。

### 3.4 设置页格式说明独占一行

**问题**：格式说明 `mp4 — 兼容性最佳…` 是一个独立的 `<div>` 带 `borderBottom`，在 Section 卡片里看起来像一条孤立的文字行。

**修复**：删掉独立 `<div>`，把说明文字移到"默认格式" SettingRow 的 `hint` 参数中。

### 3.5 历史记录导入/导出说明太占位

**问题**：一整行卡片显示"导出保存为 JSON 备份文件…"，页面视觉很重。

**修复**：改为按钮 `title` 属性（原生 HTML tooltip），hover 1 秒后自然出现。既保留了信息，又不占空间。

> **最快上手**：跑 `npm run dev` 打开浏览器 → 看看设置页的 18 项 → 切主题色看全链路联动 → 点关于页看新设计 → 然后做真机验证（扫码登录 + BV 解析）。
>
> **新增 Store 字段 7 个**：部分只有 UI 还没接后端（文件名模板、弹幕字幕、通知），见 §六优先级 3。
>
> **关于页的 CollapsibleCard**：是一个可复用的通用组件，在其他地方需要展开/折叠功能时可以直接用。
