## 四、设计决策与问题解决

### 4.1 项目命名决策

**原始名称**：BilibiliDown（和原始 Java 项目同名）
**改为**：BibiliDown

- 与原始 Java 项目 `nICEnnnnnnnLee/BilibiliDown` 区分开
- `Bibili` = B站（哔哩哔哩）+ Down（下载），简洁好记
- 文件夹名恰好是 `BibiliDown`（原始就拼成了这样，算是巧合）

**影响**：localStorage key 改名导致旧数据丢失，但用户量尚小，可接受。

### 4.2 图标设计

**方案演进**：
1. 最初：粉色方块 + "B" 文字 → 太简陋，像占位符
2. 第二版：粉红渐变圆角方块 + 白色下载箭头 → 简洁但有辨识度
3. **最终版**：同上，但调整比例更协调

**注意**：如果将来要打包 Electron，需要把 SVG 转成 `.ico` / `.icns` 格式。可以写个脚本自动生成。

### 4.3 QR 码刷新

**问题**：`LoginPanel.tsx` 中 `handleRefresh` 调用 `window.location.reload()`，用户扫完码刷个新要等整个页面重载。

**解决**：引入 `qrVersion` 状态计数器。`handleRefresh` 改为 `setQrVersion(v => v + 1)`，useEffect 依赖 `qrVersion`，自动重新生成二维码。干净利落。

### 4.4 QR 码颜色亮暗自适应

**问题**：QR 码硬编码 `dark: '#ffffff'`（白色码点），亮色主题下背景也是浅色，二维码几乎看不见。

**解决**：读取 `useUserPrefsStore` 的 `theme`，亮色用 `#1e293b`（深灰），暗色用 `#ffffff`（白色）。useEffect 依赖加上 `theme`，切换主题自动刷新二维码。

### 4.5 下载路径：浏览器 vs Electron

**问题**：浏览器 `npm run dev` 下没有 `window.electronAPI`，文件夹选择按钮不可用。

**解决**：
- Electron：`api.selectDirectory()` 弹出原生文件夹对话框
- 浏览器：使用 File System Access API 的 `showDirectoryPicker()`（Chromium 86+）
- 两种情况下都保留手动输入路径的文本框

### 4.6 版本号更新

`package.json` 中 `version: "7.0.0"` 与实际进度不符（已到第十一轮）。改为 `7.2.0`，同步更新 `App.tsx`、`StatusBar.tsx`、`tokens.css` 中的版本引用。

### 4.7 设置页 Section 标题

**之前**：`fontSize: text-body-sm` (~13px)，灰色，全大写 + 字母间距。中文大写无意义，字号太小显得"缩在角落"。

**之后**：`fontSize: text-body` (~14px)，`--text-primary` 色，`fontWeight: 600`，左侧 3px 主题色竖条做视觉锚点。层次清晰。

### 4.8 zustand Selector 返回对象导致无限渲染

**已由第十轮修复**：`useUserPrefsStore((s) => ({ loginName: s.loginName, loginFace: s.loginFace }))` 每次返回新对象 → 无限重渲染。拆成两个独立原始类型 selector 解决。本轮未再触发。
