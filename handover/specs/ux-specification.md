## 1. 设计目标与原则

### 1.1 核心设计目标

| 目标 | 说明 |
|---|---|
| **零弹窗** | 所有常规操作在单一窗口内完成，弹窗仅用于不可逆操作的确认 |
| **一眼看全** | 解析结果、下载进度、历史记录在同一个视图内可及 |
| **默认即最优** | 记住用户偏好，90% 场景只需 Enter 两次（粘贴 → 下载） |
| **反馈无处不在** | 每个操作都有即时、明确、非侵入的视觉反馈 |

### 1.2 设计原则

```
1. 输入即解析  —— 不做多余的"确认解析"按钮
2. 卡片而非弹窗 —— 解析结果内联展示，可堆叠、可关闭、可回溯
3. 右侧面板常驻 —— 下载进度全局可见，不与主内容争夺注意力
4. 芯片而非按钮 —— 选项类交互使用 Radio Chip，视觉轻量、语义明确
5. 错误就地展示 —— 错误信息展开在任务行下方，不打断用户操作流
6. 偏好自动记忆 —— 清晰度、格式、字幕偏好跨会话保持
```

## 3. 区域详细划分

### 3.1 标题栏（32px 高）

- **去原生装饰**，使用自定义标题栏
- 左侧：应用图标 (20×20) + 应用名 "BilibiliDown"
- 右侧：最小化、最大化、关闭按钮（Windows 11 风格，hover 时有圆角背景色）
- **拖拽行为**：拖拽标题栏任意位置可移动窗口（关闭/最小化按钮除外）
- **双击行为**：最大化/还原

### 3.2 导航侧栏（56px 宽，常驻）

```
┌──┐
│📥│  下载主页  ← 默认选中
├──┤
│⚙│  设置
├──┤
│📋│  历史记录
├──┤
│ℹ │  关于
└──┘
```

- 图标风格：Segoe Fluent Icons（描边风格，24×24）
- 选中态：左侧 3px 强调色竖线 + 图标着色为强调色
- 未选中态：图标灰色（`#8A8A8A`），hover 时变为深色（`#333` 亮色模式 / `#CCC` 暗色模式）
- 底部固定：设置、关于图标固定在侧栏底部
- **工具提示**：hover 1 秒后显示 tooltip（纯文字，小字号）

### 3.3 主内容区（弹性，占据剩余空间）

主内容区分三个垂直区域：

#### 3.3.1 输入栏区域（固定顶部，约 56px）

- 全宽输入框，带 placeholder "粘贴 B 站链接或 BV/av/ss/ep/md 号..."
- 右侧「解析」按钮（主要按钮，强调色填充）
- 下方 4px：自动识别标签（淡入动画显示）

#### 3.3.2 内容区域（弹性滚动）

- **单视频模式**：视频详情卡片 + （可选）剧集列表
- **批量模式**：多个视频详情卡片纵向堆叠，每个可独立操作
- **设置页面**：分组表单（下载、外观、网络、关于）
- **历史页面**：列表，每条可点击重新解析

#### 3.3.3 空状态

```
┌──────────────────────────────────┐
│                                  │
│          📺 (大图标, 64px)       │
│        粘贴一个 B 站链接开始下载  │
│                                  │
│    支持: BV号 / av号 / ep / ss   │
│          md / ml / 合集链接       │
│                                  │
└──────────────────────────────────┘
```

### 3.4 下载面板（320px 宽，右侧，可收起）

**三种显示模式**：

| 模式 | 触发条件 | 行为 |
|---|---|---|
| **常驻展开** | 窗口宽度 ≥ 1280px 且用户未手动收起 | 固定在右侧，320px |

| **覆盖展开** | 点击浮动按钮 | 从右侧滑入，覆盖在主内容上方（带半透明遮罩） |

```

                    │ PARSING  │

                    └────┬─────┘

                    ┌──────────┐
                    │ PREVIEW  │
                    │ 预览视频  │ ◀──────── 网络恢复
                    │ 信息卡片  │

                    │          │

                    └──────────┘
```

### 4.2 输入 → 解析的详细交互

```

        ├─ 输入框内容实时展示（含清除按钮 ✕）
        │
        └─ 如果启用"自动检测剪贴板"：

        │
        ▼

        │
        ├─ 解析按钮变为 loading spinner
        ├─ 输入栏下方出现"识别标签"骨架屏
        │  ┌─────────────────────────────┐
        │  │ ◌ 识别中...                  │
        │  └─────────────────────────────┘
        │
        ├─ 主内容区出现视频详情卡片骨架屏
        │  ┌────────────────────────┐
        │  │ [████████████] 封面占位  │
        │  │ ██████████ 标题闪烁      │
        │  │ ██████ UP主名闪烁        │
        │  └────────────────────────┘
        │
        ▼ (解析完成, ~200-800ms 后)
        │
        ├─ 骨架屏 300ms ease-out 动画过渡为真实卡片
        │
        ├─ 识别标签变为实体：
        │  ┌─────────────────────────────┐
        │  │ 🎬 单视频 · ⏱ 12:30 · 👤 UP主名 │
        │  └─────────────────────────────┘
        │
        └─ 焦点自动移到清晰度默认选项上（键盘用户可直接 ←→ 切换后 Enter 下载）
```

## 5. 组件规格

### 5.1 输入栏（InputBar）

```
 ┌──────────────────────────────────────────────────────┐
 │ 🔗  粘贴 B 站链接或 BV/av/ep/ss/md 号...      [解析] │
 └──────────────────────────────────────────────────────┘
    ↑ 24px icon                              ↑ 80px 按钮
```

| 属性 | 值 |
|---|---|
| 高度 | 44px |
| 圆角 | 8px |
| 背景 | 亮色: `#FAFAFA` 带 1px `#E0E0E0` 边框；暗色: `#2A2A2A` 带 1px `#444` 边框 |
| 聚焦态 | 边框变为强调色，外发光 `0 0 0 3px rgba(强调色, 0.12)` |
| 输入文字 | 14px，常规字重 |
| 前缀图标 | 24px 链接图标，颜色与 placeholder 文字一致 |
| 清除按钮 | 有内容时在输入框右侧出现 ✕，20×20 |
| 解析按钮 | 80×36px，强调色背景，白色文字，圆角 6px，字重 500 |

**自动检测剪贴板**的提示标签（在输入栏下方出现）：

```
┌──────────────────────────────────────────────────────┐
│ 💡 检测到剪贴板中的链接: bilibili.com/video/BV1...   │
│    Tab 采纳  ·  ✕ 忽略                               │
└──────────────────────────────────────────────────────┘
```

| 属性 | 值 |

| 阴影 | `0 2px 8px rgba(0,0,0,0.08)` (亮色) / `0 2px 8px rgba(0,0,0,0.3)` (暗色) |
| 边框 | 1px `rgba(0,0,0,0.06)` (亮色) / 1px `rgba(255,255,255,0.06)` (暗色) |
| 封面圆角 | 8px |
| 卡片出现动画 | 从 `opacity:0; transform:translateY(12px)` 到 `opacity:1; transform:translateY(0)`，300ms ease-out |
| 卡片关闭动画 | `opacity:1→0; max-height:400px→0; margin-bottom:16px→0`，250ms ease-in，关闭后卡片从 DOM 移除 |

**多卡片堆叠间距**: 16px

### 5.3 清晰度芯片选择器（QualityChipGroup）

这是本次设计中最核心的交互组件。


## 6. 异常流程处理

### 6.1 网络断开

**检测方式**：
- 解析请求超时（5 秒无响应）
- 下载过程中连续 3 次 socket 超时
- 主动 ping 心跳每 30 秒一次

**表现**：

```
输入栏区域 → 无变化

识别标签区域 → 变为警告标签：
┌──────────────────────────────────────────┐
│ ⚠ 网络连接异常 · 请检查网络后重试        │
└──────────────────────────────────────────┘

下载面板 → 所有进行中的任务显示为"暂停-等待网络"状态：
┌──────────────────────────────────────────┐
│ ⏳ 视频标题                              │
│ 等待网络恢复... 将在恢复后自动继续        │
│ [🔄 立即重试] [✕]                       │
└──────────────────────────────────────────┘

状态栏 → 
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 🔴 网络异常 · 下载队列已暂停 · 14:32
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**恢复行为**：
- 网络恢复后，状态栏自动变回绿色 "就绪"
- 暂停的任务自动恢复下载（不倒回已下载进度）
- Toast 通知（如果窗口不在前台）："网络已恢复，下载继续"

### 6.2 解析失败

**失败原因分类**：

| 原因 | 错误信息 | 用户可操作 |
|---|---|---|
| 网络超时 | "请求超时，请检查网络连接" | [重试] |
| 链接无效 | "无法识别该链接，请检查是否正确" | [修改链接] |
| 视频不存在 | "该视频不存在或已被删除" | 查看历史记录中的类似链接 |
| 需要登录 | "该内容需要登录后才能访问" | [去登录] |
| 地区限制 | "该内容在您的地区不可用" | 仅展示信息 |
| 服务端错误 | "B 站服务器繁忙，请稍后重试" | [重试] |

**表现**：

解析失败后，卡片区域显示错误卡片（替代视频卡片）：

```
┌──────────────────────────────────────────┐
│ 🔴                                       │
│                                          │
│  该视频不存在或已被删除                    │
│                                          │
│  可能的原因: 链接有误、视频已下架、        │
│  UP主已删除该稿件                          │
│                                          │
│  [检查链接]  [返回到输入]                │
│                                     [✕]  │
└──────────────────────────────────────────┘
```

**不弹 Modal 对话框。** 错误在卡片区域就地展示，用户可以：
- 关闭错误卡片，继续做其他事
- 修改输入框内容后重新解析
- 错误卡片在 3 分钟后自动收起为一行摘要（不占用过多空间）

### 6.3 下载失败

**失败原因分类**：

| 原因 | 检测方式 | 自动重试？ | 重试策略 |
|---|---|---|---|
| 网络超时 | SocketTimeoutException | 是 | 最多 3 次，间隔 5s/10s/30s 递增 |
| 链接失效 (403/404) | HTTP 状态码 | 是 | 重新查询下载链接后重试 1 次 |
| 磁盘空间不足 | IOException | 否 | 提示用户清理空间 |
| 文件写入失败 | IOException | 否 | 提示检查文件权限 |
| FFmpeg 合并失败 | 进程异常退出 | 否 | 保留分片文件，提示手动处理 |

**表现**（已在 5.4 组件中详述下载队列项的各种状态）。

额外细节：

**磁盘空间不足的特殊处理**：
- 下载面板顶部出现横幅警告：
  ```
  ┌──────────────────────────────────┐
  │ ⚠ 磁盘空间不足 (剩余 128MB)      │
  │ 下载队列已暂停 · [更改保存路径]   │
  └──────────────────────────────────┘
  ```
- 所有进行中的任务暂停
- 状态栏变为橙色警告色
- 用户更改保存路径或清理空间后，可手动恢复

### 6.4 FFmpeg 缺失/异常

首次启动时检测 FFmpeg：

```
如果未检测到 FFmpeg：

主内容区显示帮助卡片（非阻塞）：
┌──────────────────────────────────────────┐
│ ⚙ FFmpeg 未找到                          │
│                                          │
│ FFmpeg 用于合并视频和音频轨道。            │
│ 没有它，您将只能下载分离的视频/音频文件。   │
│                                          │
│ [自动下载 FFmpeg]  [手动指定路径]  [跳过] │
└──────────────────────────────────────────┘
```

- 不阻止用户使用 App
- 下载选项中的格式自动调整为不需要合并的格式
- 状态栏显示橙色 "⚠ FFmpeg 缺失"

### 6.5 登录过期

- 当 API 返回 401/需要登录的响应时：
  - 下载面板中该任务变为 "⚠ 需要重新登录"
  - 状态栏登录状态变为 "🔴 登录已过期 · 点击重新登录"
  - **不弹窗打断用户**
  - 用户点击状态栏的登录状态文字 → 打开登录侧边面板

### 6.6 长时间空闲

- App 空闲 5 分钟无操作 → 无任何变化
- 下载队列全部完成且空闲 30 分钟 → 可选："全部下载完成"的通知（可在设置中关闭）
- 不主动最小化、不进入省电模式（下载工具不应自行休眠）

### 7.2 数据持久化

| 数据 | 持久化位置 | 说明 |
|---|---|---|
| 用户设置 | `config/settings.json` | 包括清晰度偏好、保存路径、外观设置 |
| 下载历史 | `config/history.json` | 最近 500 条已下载记录（BV号+标题+时间） |
| 输入历史 | `config/input-history.json` | 最近 100 条输入链接 |
| 登录状态 | `config/cookies.json` | 加密存储的 Cookie |
| 任务队列 | 内存（不持久化） | App 重启后清空——符合下载工具预期 |

**设计决策：任务队列不跨会话保留。**
- 原因：用户重启 App 时有明确的"重新开始"的心智模型
- 例外：如果用户在设置中开启了"记住下载队列"，则退出时将未完成队列序列化，下次启动恢复（每个任务提示是否继续）

### 7.3 动画时长规范

| 动画 | 时长 | 缓动函数 |
|---|---|---|
| 卡片出现 | 300ms | `cubic-bezier(0.0, 0.0, 0.2, 1.0)` (ease-out) |
| 卡片关闭 | 250ms | `cubic-bezier(0.4, 0.0, 1.0, 1.0)` (ease-in) |
| 登录面板滑入/出 | 300ms | `cubic-bezier(0.4, 0.0, 0.2, 1.0)` |
| 下载面板展开/收起 | 250ms | `cubic-bezier(0.4, 0.0, 0.2, 1.0)` |
| 芯片选中态切换 | 150ms | `cubic-bezier(0.4, 0.0, 0.2, 1.0)` |
| 进度条更新 | 200ms | `ease-out` |
| 骨架屏闪烁 | 1500ms 循环 | `ease-in-out` |
| Toast 通知出现 | 200ms | `ease-out` |
| Toast 通知消失 | 200ms after 4000ms delay | `ease-in` |
| Hover 状态变化 | 100ms | `ease` |

- 所有交互元素可通过键盘操作（Tab 导航顺序：输入框 → 芯片 → 下载按钮 → 下载面板）
- 高对比度模式下，用更粗的边框替代阴影/透明度

| 12 | 每个 Tab 独立管理生命周期 | 🟡 中 | 统一的内容区，卡片堆叠 | 最多同时展示 5 张卡片，超过的折叠 |
